# Handoff: P1 (after P1.2)

## What was completed

- Preserved the accepted P0/P1.1 state, verified the refactor branch and both legacy baseline refs,
  and found no authority-document conflict for P1.2.
- Added strict `ReferenceCanvasCandidate` and `ReferenceCanvasDecision` contracts with stable
  candidate-set hashing and explicit T1--T4 spatial eligibility.
- Implemented deterministic reference selection independent of input order:
  authoritative official/human native mask first, then complete registered mask grids by finest
  known GSD with lexical modality tie-break, then the sole original language image.
- Made ambiguous multi-grid coverage/GSD cases fail closed instead of selecting a runtime fallback.
- Strengthened `TransformStep` and Canonical Parent v3 schema contracts for exact identity,
  crop, resize and pad parameters, non-empty continuous chains, endpoint binding and coordinate
  inverse availability.
- Froze raster replay as pixel-edge coordinates, half-pixel-center sampling and clamp borders;
  HWC images use bilinear while HW mask/valid rasters use nearest only.
- Implemented dependency-free CPU reference operations for crop/resize/pad, forward/inverse points
  and boxes, covering integer box quantization, explicit HWC/HW shape contracts and zero padding.
- Defined inverse availability on the retained valid-content footprint. Padded coordinates raise an
  error instead of fabricating a source-space inverse.
- Implemented effective-target construction as `mask AND valid`; padding and nodata are excluded
  from later loss, metric and pooling populations by contract.
- Enforced exactly one reference-aligned modality per parent. Unregistered support may be
  `global_only`, but cannot carry pixel-level transforms.
- Implemented coverage-preserving reference half-open to Qwen integer `[0,1000]` conversion.
  Round-trip expansion is bounded per edge by `ceil(reference_axis_size/1000)`.
- Added synthetic tests covering selection priority/order invariance, ambiguity, spatial eligibility,
  transform continuity/round-trip, bilinear versus nearest, padding/nodata exclusion, global-only
  support and Qwen conversion across representative canvas scales.
- Ran the complete P1 suite: 32/32 tests passed. Two independent synthetic trace builds produced
  SHA-256 `d5bcd5e9ed6c9a2b5b93ec1d96b038bb893b033e56ac19dbbbf8ddb5f0e92d09`.
- Committed the implementation locally as `ac4be61e6fd994408934885095563311b9e43ebe`.

## Known risks

- The CPU nested-sequence raster path is the deterministic P1.2 policy oracle, not the eventual
  high-throughput source materializer. A later backend must prove equivalent geometry/resampling.
- Qwen integer normalization is necessarily quantized; it preserves coverage but can expand each
  edge by the report's declared bound.
- JSON Schema cannot compare transform-chain endpoints or cross-reference a modality ID inside an
  array. Pydantic is the authoritative cross-field validation layer for those invariants.
- Source paths, group identities, mask layouts, nodata conventions and transforms remain unverified
  until a scoped live raw-source structure audit.
- Source availability does not imply license eligibility. All live configs remain fail-closed.
- P1 is not accepted and no Small benchmark exists.
