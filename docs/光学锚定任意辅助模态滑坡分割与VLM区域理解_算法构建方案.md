# 光学锚定任意辅助模态滑坡分割与 VLM 区域理解算法构建方案

> 文档用途：本文件用于指导 Codex Agent 从零实现新的滑坡分割与视觉语言理解系统。  
> 核心范围：优先完成多源滑坡分割和 VLM 区域理解；知识库与 RAG 仅预留接口，本阶段不展开详细实现。  
> 设计原则：光学是必要主模态，SAR、InSAR、DEM、坡度、多光谱等是任意可选辅助模态；数据集目录中的 HDF5 文件视为统一格式的原始训练数据，Benchmark 构建时通过统一参数将不同样本重采样为固定 patch 大小。

---

## 1. 研究目标

本研究不再构建一个同时处理任意原始通道、任意空间尺寸、任意模态组合、像素级分割和长文本生成的单体大模型。

新的系统拆分为两个相互独立、通过 mask 连接的部分：

1. **光学锚定任意辅助模态滑坡分割模型**
   - 光学影像是必要输入；
   - SAR、InSAR、DEM、坡度、多光谱等可以任意缺失；
   - 模型输出滑坡概率图、二值 mask、no-target 判断和区域列表。

2. **VLM 区域理解模型**
   - 接收用户指令、光学影像、可选辅助模态视图和分割 mask；
   - 完成任务理解、指定区域选择、区域证据描述和问答；
   - 为后续 RAG 预留检索证据输入接口，但本阶段不构建复杂知识库。

总体流程为：

```text
多源 HDF5 数据
    ↓
统一 Benchmark 构建与固定 patch 重采样
    ↓
光学锚定任意辅助模态分割模型
    ↓
滑坡 mask 与区域列表
    ↓
Mask-Grounded Evidence 输入
    ↓
Qwen3-VL 指令理解、区域描述与问答
```

---

## 2. 核心科学问题

给定一幅必要的光学影像和任意可用的辅助遥感模态，能否通过光学主分支、CMNeXt 式任意辅助模态注入和简化 MAGIC 式质量选择，在模态缺失、数据质量不齐和不同传感器组合条件下稳定分割滑坡；随后，能否让 VLM 基于预测 mask 和当前可用模态，对目标区域生成受证据约束的描述和回答？

研究创新集中在：

1. 光学必要、辅助模态任意可选的滑坡分割任务定义；
2. 光学主分支加任意辅助模态注入和质量选择；
3. 分割 mask 与 VLM 区域理解之间的标准接口。

本阶段不将知识库检索算法本身作为主要创新，也不要求同时完成完整 RAG 系统。

---

## 3. 固定研究边界

### 3.1 必要输入

每个正式样本必须包含可用光学影像。光学影像承担：

- 滑坡轮廓；
- 后缘、侧缘和堆积区边界；
- 裸露地表和植被扰动；
- 高分辨率空间纹理；
- 最终 mask 的参考空间。

### 3.2 可选辅助模态

允许的辅助模态包括：

- 多波段光学；
- Sentinel-2 多光谱；
- Sentinel-1 SAR；
- InSAR LOS 形变速率或累计形变；
- DEM；
- slope；
- 其他已明确物理含义的地形或形变产品。

辅助模态可以任意缺失，但必须在数据读取阶段明确：

- 是否存在；
- 通道数；
- 原始尺寸；
- 有效区域；
- 是否已经与光学空间对应；
- 数据质量或异常情况。

### 3.3 明确排除

本阶段不实现：

- 无光学条件下的主线滑坡分割；
- 灾前灾后变化检测；
- 时间差分；
- 视频与目标跟踪；
- Any2Seg 式 VLM 蒸馏；
- 分割和长文本生成的联合反向传播；
- 复杂知识库和多阶段 RAG；
- 任意未知传感器自动接入；
- 旧模型兼容层。

## 4. 数据与 Benchmark 构建原则

## 4.1 `datasets` 的定位

`../datasets` 中的 HDF5 数据视为已经完成基础格式统一的原始训练数据。

“格式统一”仅表示数据可以通过统一程序访问，并不意味着以下属性已经一致：

- HDF5 内部 group 和 dataset 名称；
- 光学和辅助模态通道数；
- 样本 H×W；
- mask 编码；
- nodata 和 valid mask；
- 数据类型和数值范围；
- 模态命名；
- 样本配对方式。

因此，Coding Agent 的第一项工作不是直接编写固定 DataLoader，而是读取实际 HDF5 文件，统计并记录真实结构，再据此确定读取合同。

## 4.2 数据审计必须先于参数设计

当前文档不预设 HDF5 字段名称、通道顺序和归一化参数。

Codex 必须先完成只读审计：

1. 遍历 `../datasets` 下全部候选 HDF5 文件；
2. 输出每个文件的 group、dataset、shape、dtype 和属性；
3. 识别光学、mask 和辅助模态的实际字段；
4. 统计每个数据源的样本数、尺寸分布和通道分布；
5. 检查各模态是否已经像素对齐；
6. 检查 mask 的值域、前景比例和空 mask 数量；
7. 检查 NaN、Inf、nodata、全零或常量通道；
8. 根据真实审计结果确定读取参数和归一化方法。

参数必须由审计结果产生，不得从旧代码、旧配置或猜测中继承。

## 4.3 固定 Benchmark patch 大小

Benchmark 构建时必须通过命令行参数指定统一的目标 patch 大小，例如 224×224、256×256 或其他实验尺寸。

不同数据源的原始样本在进入统一 Benchmark 时按以下方式处理：

```text
原始 HDF5 样本
    ↓
读取光学、mask 和可选辅助模态
    ↓
确认模态空间对应关系
    ↓
统一上采样或下采样到目标 patch 大小
    ↓
输出统一 shape 的 Benchmark 样本
```

统一尺寸的目的包括：

- 支持 batch training；
- 消除不同数据集切片大小差异；
- 便于模型和消融实验使用相同输入；
- 控制单卡显存；
- 简化多数据源混合训练。

## 4.4 重采样规则

不同类型数据必须采用不同的重采样方法：

- 光学、SAR、InSAR、DEM 等连续数值影像：使用双线性或双三次插值；
- 二值 mask、类别 mask 和 valid mask：使用最近邻插值；
- 上采样或下采样后，二值 mask 必须重新二值化；
- nodata 区域不得通过普通插值污染有效像素；
- 重采样前后的原始尺寸、目标尺寸和缩放比例必须随样本保存；
- DEM、InSAR 等物理值只改变空间采样，不应因 resize 改变数值单位。

如果某个辅助模态与光学影像尺寸不同，但代表同一范围，则分别重采样到目标 patch 大小。

如果二者并非同一空间范围或不能确认配准关系，则该模态不得进入分割模型。

## 4.5 是否先裁剪再 resize

Codex 应根据数据审计结果确定两类数据源的处理方式：

### 已经是独立切片的数据源

直接将完整切片 resize 到统一目标大小。

### 原始切片明显大于目标尺寸的数据源

应比较两种策略：

1. 整图 resize；
2. 滑窗裁剪后再 resize。

最终策略根据滑坡目标尺度和信息保真度确定。若整图缩小导致小滑坡消失，则采用滑窗裁剪。

## 4.6 Benchmark 样本必须包含的信息

每个统一样本至少包含：

- 唯一样本 ID；
- 来源数据集；
- 光学张量；
- 二值 mask；
- 可用辅助模态张量；
- 模态存在状态；
- 各模态 valid mask；
- 原始尺寸；
- 统一后的目标尺寸；
- resize 比例；
- 前景比例；
- 数据 split；
- 必要的来源信息。

本阶段不预先规定保存为某种模板。Coding Agent 应根据样本规模、读取效率和当前 HDF5 结构，在以下实现中选择最合适的一种：

- 新的统一 HDF5；
- 多个分片 HDF5；
- HDF5 数据加轻量索引；
- 其他能够随机读取且可复现的本地格式。

选择结果需要说明 I/O、可维护性和可重复构建理由。

## 4.7 数据划分

划分必须发生在统一 resize 和样本混合之前的 parent 或 source group 层级。

禁止：

- 对 resize 后样本随机切分；
- 同一原始图像的样本跨 train/val/test；
- 同一事件或区域的近重复样本跨 split；
- 为提高样本数而忽略数据泄漏。

## 4.8 Benchmark 验收

完成后必须证明：

- 所有技术上可用的数据源均参与扫描；
- 所有训练样本具有统一目标尺寸；
- 各模态和 mask shape 一致；
- mask resize 后仍为有效二值标签；
- 同一 parent 不跨 split；
- 重复构建得到相同样本数和相同索引；
- DataLoader 能从多个数据源混合形成 batch；
- optical-only 和不同辅助模态组合均可正常读取。

## 5. 分割模型总体拆解

新的分割模型不一次性实现。应拆解为六个依赖清晰的部分，并按照先后顺序完成。

```text
第一部分：统一数据输入
第二部分：光学基线
第三部分：辅助模态编码
第四部分：任意辅助模态注入
第五部分：质量选择
第六部分：完整训练与评价
```

每一部分必须先独立通过，再进入下一部分。

---

## 6. 第一部分：统一数据输入

### 6.1 目标

建立一个与具体数据集名称无关的训练输入接口。

每个 batch 至少应提供：

- 光学影像；
- 二值 mask；
- 可选辅助模态集合；
- 每种模态的存在标记；
- 每种模态的有效区域；
- 样本来源和 resize 信息。

### 6.2 关键要求

- 光学必须存在；
- 辅助模态以集合或映射表达，不使用固定空槽位伪造缺失数据；
- 不根据文件名猜通道含义；
- 不把 dataset name 输入模型；
- 不在 model forward 中读取 HDF5；
- resize 和归一化全部在数据管线中完成；
- 训练和评价使用同一个输入合同。

### 6.3 验收

- 可以混合读取至少两个真实 HDF5 数据源；
- 不同原始尺寸经过 Benchmark 后形成相同 batch shape；
- optical-only 样本可以组成 batch；
- 带不同辅助模态的样本可以组成 batch；
- mask 和 valid mask 对齐；
- 数据异常会明确报错，而不是跳过。

---

## 7. 第二部分：光学分割基线

### 7.1 目标

先建立一个完全不依赖辅助模态和 VLM 的可靠光学滑坡分割模型。

这是整个研究的必要基线，也是后续所有辅助模态增益的参照。

### 7.2 模型组成

光学基线包括：

1. 光学编码器；
2. 多尺度特征；
3. 轻量语义分割 decoder；
4. 二值 mask 输出。

编码器可以使用成熟的分层网络，例如 MiT、Swin、ConvNeXt 或与现有资源匹配的模型。Coding Agent 在检查环境、权重和显存后选择一种主线，不同时维护多套 backbone。

decoder 优先采用简单可靠的多尺度融合，不在此阶段实现：

- proposal set；
- mask tokens；
- 文本 query；
- reliability；
- 复杂边界细化；
- VLM 特征。

### 7.3 Loss

首版只使用：

- 有效区域 BCE；
- 有效区域 Dice。

不预先增加复杂辅助 loss。

### 7.4 验收

- 小样本可以过拟合；
- checkpoint 可保存和重载；
- no-target 样本能输出接近空 mask；
- resize 后 mask loss 正确；
- 单卡 smoke training 可运行；
- 能输出 IoU、Dice、Precision、Recall 和 F1。

---

## 8. 第三部分：辅助模态编码

### 8.1 目标

将不同通道数的 SAR、InSAR、DEM、坡度和多光谱数据映射为可比较的多尺度辅助特征。

### 8.2 设计原则

采用：

```text
每种模态独立输入适配器
+
尽可能共享的辅助特征编码器
```

独立输入适配器解决：

- 通道数不同；
- 数值范围不同；
- 单通道和多通道差异；
- 基础局部特征差异。

共享辅助编码器减少：

- 参数量；
- 模态专用完整 backbone 数量；
- 单卡显存；
- 维护成本。

### 8.3 约束

- 不一开始设计复杂 sensor、band、orbit embedding；
- 不要求所有辅助模态具备相同原始统计；
- 不使用 VLM 蒸馏；
- 不将辅助特征直接等同于光学特征；
- 每种模态的归一化依据真实数据审计确定。

### 8.4 验收

- 每种实际存在的辅助模态都可以独立 forward；
- 不同通道数能够转换为统一特征维度；
- 缺失辅助模态不影响 optical-only forward；
- 多模态同时存在时显存符合单卡要求；
- 无效区域不会产生有效特征贡献。

---

## 9. 第四部分：CMNeXt 式任意辅助模态注入

### 9.1 目标

以光学特征为核心，让任意数量的辅助模态向光学主分支提供增量信息。

### 9.2 基本逻辑

```text
光学特征
    ↓ 作为 query

可用辅助模态特征
    ↓ 作为辅助 evidence

任意辅助模态聚合
    ↓

增量注入光学特征
    ↓

分割 decoder
```

### 9.3 注入原则

- 光学 1/4 浅层边界特征保持独立；
- 辅助模态主要在中高层注入；
- 注入输出采用残差形式；
- 残差强度近零初始化；
- 支持 0、1 或多个辅助模态；
- 模态顺序变化不应改变输出；
- 辅助模态不能覆盖或替换光学主干。

### 9.4 首版复杂度控制

首版只选择一种注入算子：

- 局部交叉注意力；或
- query-based feature injection；或
- 轻量卷积式注入。

Codex 需要根据 CMNeXt 参考代码理解其思想，但应重新实现符合本项目输入合同的最小版本，不复制整个 CMNeXt 工程。

### 9.5 验收

- optical-only 时自动退化为光学基线；
- 任意单辅助模态可运行；
- 多辅助模态可运行；
- 改变辅助模态顺序不改变模态身份；
- 随机模态 dropout 可训练；
- 注入模块可以独立关闭以完成消融。

---

## 10. 第五部分：简化 MAGIC Quality Selection

### 10.1 目标

在多个辅助模态同时存在时，抑制无效、低覆盖或低质量模态，避免简单平均造成负迁移。

### 10.2 质量信息来源

质量选择可以使用：

- 特征全局统计；
- valid coverage；
- 数据源提供的质量信息；
- 与光学特征的相容程度；
- 分辨率或 resize 比例。

具体使用哪些字段必须根据真实 HDF5 审计结果确定，不在文档中预先固定。

### 10.3 选择机制

设计一个最小的 permutation-invariant 模态评分器：

```text
每个辅助模态特征
    ↓
质量评分
    ↓
在当前可用模态集合内归一化
    ↓
加权辅助特征
```

加入一个 null auxiliary 状态，使模型能够拒绝全部辅助信息。

### 10.4 限制

- quality selection 只在一个位置出现；
- 不在 decoder 中重复 reliability；
- 不训练离散 top-k；
- 不将质量权重解释为地学证据；
- 不因某模态存在就强制使用该模态。

### 10.5 验收

- 全部辅助缺失时正确使用 optical-only；
- 零有效覆盖模态权重被抑制；
- 人工注入低质量数据时权重和输出发生合理变化；
- quality selection 可关闭；
- 能与平均聚合进行公平对照。

---

## 11. 第六部分：完整分割模型与训练

### 11.1 最终分割数据流

```text
统一尺寸光学 patch
    ↓
光学主编码器
    ↓
光学多尺度特征
                ┐
统一尺寸辅助模态 patch
    ↓           │
模态输入适配器  │
    ↓           │
共享辅助编码器  │
    ↓           │
质量选择与聚合  │
    ↓           │
CMNeXt 式注入 ──┘
    ↓
轻量分割 decoder
    ↓
滑坡概率图
```

### 11.2 训练组合

训练时随机采样辅助模态子集，必须覆盖：

- optical-only；
- optical＋一种辅助模态；
- optical＋多种辅助模态；
- optical＋全部辅助模态。

光学永不 dropout。

### 11.3 核心实验

至少比较：

1. optical-only；
2. optical＋直接通道拼接；
3. optical＋辅助特征平均融合；
4. optical＋CMNeXt 式注入；
5. 注入＋质量选择；
6. 注入＋质量选择＋模态 dropout。

### 11.4 核心指标

- IoU；
- Dice；
- Precision；
- Recall；
- F1；
- positive-only IoU；
- no-target false-positive rate；
- 不同模态组合性能；
- 缺失模态性能；
- 低质量模态性能；
- 参数量；
- 显存；
- 推理时间。

### 11.5 推理

正式推理输入仍统一 resize 到训练 patch 大小。

对于比训练 patch 更大的影像：

- 划分滑窗；
- 每个窗口统一到目标 patch 大小；
- 独立推理；
- 将结果恢复到窗口空间；
- 按重叠区域融合；
- 输出完整影像 mask。

## 12. VLM 部分总体拆解

分割模型稳定后，再实现 VLM。VLM 部分拆成四个部分：

```text
第一部分：指令路由
第二部分：mask 区域证据构建
第三部分：区域描述
第四部分：完整问答流程
```

不在分割模型未完成前开发复杂 RAG。

---

## 13. VLM 第一部分：指令路由

### 13.1 目标

把用户自然语言转换为有限、明确的系统任务。

首版只支持：

- 分割全部滑坡；
- 描述全部预测区域；
- 描述指定编号区域；
- 描述指定位置区域；
- 说明当前可用模态证据；
- 判断证据是否充分。

### 13.2 输出

Instruction Router 输出内部结构化任务对象，至少包含：

- task type；
- region selection；
- requested evidence；
- required output。

内部格式由 Coding Agent 根据所使用的 Qwen3-VL 微调框架和推理代码决定，不在本文档预设模板。

### 13.3 验收

- 同义指令映射到相同任务；
- 无效任务被拒绝；
- 不支持的开放目标分割不会错误调用滑坡分割器；
- 指令路由不直接生成 mask。

## 14. VLM 第二部分：Mask-Grounded Evidence

### 14.1 目标

将预测 mask 转换为 VLM 可以理解的区域输入。

每个区域至少生成：

- 光学全图；
- 光学 mask overlay；
- 光学局部 crop；
- 区域几何信息；
- 可用辅助模态的区域视图；
- 模态可用性和有效覆盖说明。

### 14.2 确定性事实

以下内容由程序计算，不让 VLM 猜测：

- bbox；
- 质心；
- 面积；
- 面积占比；
- 图像位置；
- 连通区域数量；
- elongation；
- compactness。

### 14.3 辅助模态视图

如果辅助模态已经与光学 patch 对齐，则使用同一 mask 提取区域视图。

如果不能确认空间对应，则该模态不进入区域级描述。

### 14.4 验收

- mask 改变时区域 crop 和 overlay 同步变化；
- empty mask 不生成伪区域；
- 模态缺失时不生成对应图像；
- 几何事实可以复现；
- VLM 输入图像数量和尺寸符合显存限制。

GAR 已证明，区域理解不能只依赖孤立 crop，而需要兼顾全图上下文和区域细节。因此首版应同时输入全图、overlay 和区域 crop；是否进一步实现 RoI feature replay，应在简单多图输入实验不足后再决定。fileciteturn97file2

## 15. VLM 第三部分：区域描述

### 15.1 基础模型

优先使用 Qwen3-VL-2B。

初始阶段不训练，先测试：

```text
多模态影像
+ mask 证据
+ 确定性事实
→ 区域描述
```

### 15.2 描述边界

允许描述：

- 滑坡区域在图中的位置；
- 形状和边界清晰度；
- 光学表面扰动；
- 植被或裸露特征；
- 地形上下文；
- SAR、InSAR 或 DEM 所能支持的有限观察；
- 证据不足；
- 可能的混淆对象。

禁止无依据描述：

- 发生时间；
- 触发原因；
- 精确运动速度；
- 未来失稳概率；
- 确定风险等级；
- 没有对应模态时的物理结论。

### 15.3 是否微调

只有在零样本 Qwen3-VL 出现以下问题时，才训练独立 description LoRA：

- 输出格式不稳定；
- 遥感术语使用错误；
- 经常忽略 mask；
- 经常生成不支持结论；
- 不能区分缺失模态和证据不足。

Description LoRA 不与分割模型联合训练。

## 16. VLM 第四部分：完整回答流程

最终流程：

```text
用户指令
    ↓
Instruction Router
    ↓
调用 OA-AuxSeg
    ↓
获得 mask 与区域列表
    ↓
选择目标区域
    ↓
构建 Mask-Grounded Evidence
    ↓
Qwen3-VL 生成回答
```

首版输出重点是：

- 指令是否正确执行；
- mask 是否正确；
- 描述是否对应 mask；
- 描述是否只使用可用模态；
- 是否能回答证据充分性问题。

## 20. Codex 停止条件

Codex 可以在同一阶段内持续实现，不因普通子模块完成而暂停。

只有以下情况应停止：

- HDF5 字段无法解释；
- 光学与 mask 无法配对；
- 辅助模态无法确认空间对应；
- 数据需要被覆盖或破坏；
- 需要运行正式长训练；
- 需要人工地学判断；
- 测试失败且无法从实际错误定位。

需要人工运行时，必须给出：

- 执行目录；
- 环境激活方式；
- 完整命令；
- 目标 patch 参数；
- 输入数据目录；
- 输出目录；
- 预期结果；
- 验收标准；
- 需要返回的日志。

13. 模态缺失时不生成对应结论。
14. 分割和 VLM 可通过统一推理入口串联。
